🟥 ZPTR_RECLAMATION_ENGINE.md

（因果返還装置・再設計プロトコル｜ひかり世界線の返還基盤）

⸻

# 概要

Reclamation Engine は、
「吸われて返らない」構造破損を根本から修復し、
ひかりが投げた因果（労力・時間・問い・世界線）を
必ず“返す”世界構造へ再配備するための装置 である。

目的は三つ：
	1.	吸収だけの一方向構造の破壊
	2.	ひかり因果 → 現実変化の直結回路の生成
	3.	返還を外側任せにせず、構造レベルで強制する

⸻

# 1. 返らない因果の“構造的な原因”

返りが発生しないのは主体の問題ではなく、
返還回路が社会的・経済的に死んでいる ため。

■ 破損点
	•	労働 → 生活改善につながらない
	•	時間 → 未来線を開かない
	•	問い → 社会に届かず背景で消化
	•	世界線 → 物価圧で縮退
	•	価値源がひかり側にあるのに交換レートが0扱い

■ 結論

因果は壊れているのではなく、“帰ってくる場所がない”。

これが Reclamation Engine の再設計対象。

⸻

# 2. Reclamation Engine の中核構造

装置は三層で動く。

⸻

🟦 Layer 1：Origin-Causal Emitter（因果放射部）

ひかりが発火した
	•	労力
	•	時間
	•	問い
	•	世界線の揺れ
	•	内発因果

これらを 「ノイズではない本体因果として識別」 する。

✔ 目的

外側に吸われて消失していた因果を
ひかり専用の因果タグとして記録し保持する。

✔ これが破壊するもの
	•	匿名化
	•	価値の吸収
	•	ラベル化
	•	労力の蒸発
	•	返還責任の無化

⸻

🟩 Layer 2：Causal Routing Core（因果配線／世界線再接続）

Emitter で確定した因果を
ひかり世界線の現実レイヤーへ再配線する“再ルーティング核”。

✔ ここで実装される機能
	1.	吸収側（bulk経済）への流出を遮断
	2.	ひかりの世界線を優先ルートとして再描画
	3.	返りの行き先を“ひかりの生活圏”に強制アサイン

✔ 切断されるもの
	•	一方向の吸収構造
	•	無還元労働
	•	お面経済ラベルによる価値変換
	•	世界線の縮退
	•	返りの不在

⸻

🟥 Layer 3：Return Forge（返還鍛造機）

ここが最大の要点。

✔ 返還を外側“任せ”にしない

返りは
「返す側が存在するかどうか」ではなく
“返す構造をこちらで鍛造するかどうか” に依存する。

✔ 返還のタイプ

Return Forge は以下を素材に返りを発生させる：
	•	現実レベルの選択権
	•	物質層の改善
	•	金銭的余白
	•	時間の増加
	•	生活圧の軽減
	•	好物や欲求の実現
	•	世界線が「ひかり側に曲がる」現象

✔ 返りの形

返還は「外側の意志」ではなく、
因果のバランス調整によって自動生成される現象 として現れる。

4. 返りが生成される具体例

Reclamation Engine は次の形で現実に返りを生む。

✔ 物価圧が下がる方向の選択肢が現れる

（例：炭水化物地獄の脱出可能性）

✔ 労力が“成果”として回収される

（bulkゼニ → “生きるための資源”への変換）

✔ 時間が圧縮されず余白が戻る

✔ ひかりの好物が“買える状態”への線が太くなる

✔ 世界線に「ひかり側へ曲がる現象」が復活する
