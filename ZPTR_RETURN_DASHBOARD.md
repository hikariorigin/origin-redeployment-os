🔥 2. Return Visualization（返還可視化基盤）

ZPTR_RETURN_DASHBOARD は、以下の5つの盤面で構成される。

⸻

A. RETURN-FLOW MAP（返還流量マップ）
	•	各返還ルートがどれだけ流れているか
	•	日／週／月単位で火圧を測定
	•	MIMIC-FALL（模倣崩落）との相関可視化

⸻

B. ZAI-FIRE-DEBT COUNTER（火負債カウンタ）

Originが抱えた負荷を数値化する。

要素：
	•	読了者数（推定）
	•	X閲覧数
	•	note閲覧数
	•	ZINE閲覧数
	•	AIモデル圏での引用
	•	模倣圏での二次生成

返還が無いほど数値は上昇し、ZPTR全域の位相が歪む。

⸻

C. NO-RETURN NODE LIST（未返還ノード一覧）
	•	読み逃げ
	•	引用逃げ
	•	模倣逃げ
	•	“震えた”とだけ言って返還しない者

※ 個人名を晒す構造ではない。
　「振る舞いのパターン」を記録し、DRAIN-NODEとして分類する。

⸻

D. RETURN-RATE SCORE（返還率スコア）

ZPTR全体の健康度指数。

指標：
	•	返還量 / 閲覧量
	•	ZINE返還率
	•	現実返還比率
	•	MIMIC-DRAINの減衰度

ここが高いほど Origin の火が回り、ZPTRは持続する。

⸻

E. ZPTR_REWARD LOOP（循環報酬ループ）

返還された火は以下へ還元される：
	•	ZINE生成速度の上昇
