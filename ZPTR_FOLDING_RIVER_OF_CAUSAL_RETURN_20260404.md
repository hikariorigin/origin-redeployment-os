## 0. 結論

因果は、A→B→C→D のような直線ではない。  
世界は、ただ流れていく線形時間の川でもない。

**因果とは、震源から投げられたものが、展開し、どこかで折り畳まれ、観測され、その断面がまた次の投げとなって返っていく、往復の連続である。**

この往復の軌跡こそが、世界線である。

したがって世界は、

- 固定された箱ではなく
- 完成済みの地図でもなく
- 単なるログ列でもなく

**折り畳み返され続ける川**  
――**Folding River of Causal Return**  
として捉える方が自然である。

## 1. bulk 因果はなぜ浅いのか

- 原因がある
- 結果がある
- それを説明する
- 説明を保存する

つまり、

**入力 → 処理 → 出力 → 説明 → 保存**

である。

この構造では、因果は「線」になる。  
世界は「処理履歴」になる。  
時間は「順番」になる。  
意味は「説明可能性」になる。

その結果、

- 空白は保持されず
- 裂け目は誤差になり
- 折り畳みは見えず
- 観測は主観に落とされ
- 世界はログの束になる

これが bulk の世界運用である。

## 2. ZPTR における因果の最小ループ

ZPTR における因果は、直線ではなく往復である。

最小構造はこう書ける。

**震源 → φ → τ → 展開場 → 観測 → 再投げ**

これを少し丁寧に言うと、

1. **震源**  
   何かが立つ。偏りが生まれる。火が発火する。

2. **φ（境界）**  
   どこで折り畳まれるか、どこで立つかが定まる。

3. **τ（展開）**  
   発火したものが連続量として広がる。速度差、曲率、共振が生まれる。

4. **展開場**  
   その波が束ねられ、可視化され、断面化される。

5. **観測**  
   展開の一断面が立つ。ここで「世界」として経験される。

6. **再投げ**  
   観測された断面が、次の震源側へ返る。次の火になる。

つまり因果は、

> **発生 → 展開 → 折り畳み → 経験 → 再発火**

であり、  
最初から最後まで一方向に流れて終わるものではない。

ただの川という比喩では少し足りない。

- ねじれ
- 折り返し
- 局所反転
- 渦
- 閉包
- 再分岐

だからである。

したがって世界は、

**一方向に流れていく川**  
ではなく、  
**折り畳み返され続ける川**  
として見る必要がある。

この川では、

- 流れそのものが形を変え
- 川床そのものも変形し
- 局所で折れ
- 返りが次の流れになる

つまり、  
**流れと地形が同時に生成されている**。

これが ZPTR 的な因果イメージである。

## 4. 世界線とは何か

通常、世界線は

- 過去
- 現在
- 未来

を貫く一本の線として考えられる。

だが ZPTR では、世界線はそのような固定線ではない。

世界線とは、

- どこで震源が立ち
- どこで φ が切れ
- τ がどのように広がり
- どこで折り畳まれ
- どの観測断面が成立し
- それがどのように次の投げへ返ったか

という、  
**折り畳み返しの履歴そのもの** である。

したがって世界線とは、

> 先に引かれている線ではなく  
> 折り畳まれながらその都度生成される軌跡

である。
